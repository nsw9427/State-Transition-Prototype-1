# State-Transition-Prototype-1

This is a very basic code of Can-Sat software state transition logic without actually processing packets from the cansat
Here I have taken an array of dummy packets of just pressure data because there is no need of any other telemetry data for the state transition logic
I have used enums for better understanding of the states instead of representing them in integers
I have used switch case so that there is only a linear transition of states according to the mission requirements
